# 备份恢复说明

> English summary: Back up the private `.env` file and the full persistent data directory. Do not publish backups, logs, real photos, or database files.

生产环境至少要备份私有 `.env` 文件和完整的持久化数据目录。备份文件、日志、真实照片和数据库文件不应进入公开仓库。

## 必须备份的内容

- `.env`
- `HOST_DATA_DIR` 指向的完整数据目录
- 数据库持久化目录
- 原图、预览图、缩略图、导出文件和备份目录

## 本机示例

```text
<project-root>/data
<project-root>/.env
```

## 群晖示例

```text
/volume1/docker/engineering-photo-map/data
/volume1/docker/engineering-photo-map/.env
```

这是示例路径，不是固定要求。请按自己的 NAS 目录规划修改。

## 备份建议

1. 停止写入，或选择低峰期备份。
2. 备份 `.env` 和完整数据目录。
3. 将备份保存到不同磁盘或远程存储。
4. 定期抽查备份能否解压和读取。
5. 重大升级前先做一次完整备份。

## 恢复思路

1. 准备同版本或兼容版本代码。
2. 恢复 `.env`。
3. 恢复 `HOST_DATA_DIR` 指向的数据目录。
4. 启动 Docker Compose 或 Portainer Stack。
5. 检查 API 健康状态、项目列表、照片预览和地图显示。
