# 安全与隐私说明

> English summary: Engineering Photo Map may process photos, locations, project names, and deployment configuration. Sanitize all shared information and keep private data out of the public repository.

本项目可能处理照片、位置、项目名称和部署配置。公开或协作前请特别注意数据边界，所有共享内容都应先脱敏。

## 不要提交

- `.env` 或任何真实环境变量文件
- 真实照片、截图、缩略图、预览图
- 数据库文件或数据库持久化目录
- 日志、检查报告、备份包、导出包
- 真实域名、内网 IP、公网端口、账号、密码、token
- Portainer、NAS 或服务器控制台截图

## 部署建议

- 生产环境必须修改 `POSTGRES_PASSWORD`。
- 只允许必要的 `CORS_ORIGINS`。
- 前端 `VITE_ALLOWED_HOSTS` 只配置实际需要的域名或地址。
- 不要将 PostgreSQL 和 Redis 直接暴露到公网。
- 定期备份，并测试恢复流程。

## 提 Issue 前

请先脱敏日志和截图。不要上传真实照片、客户信息、坐标、域名、IP、端口、账号或密码。

如果需要描述网络访问问题，请使用 `photos.example.com`、`api.photos.example.com`、`<NAS_LAN_IP>`、`<PUBLIC_HTTPS_PORT>`、`<API_PORT>` 和 `<FRONTEND_PORT>` 这类占位符。
