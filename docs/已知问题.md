# 已知问题

> English summary: This page lists known limitations and common setup notes for the first public release.

本文记录首次公开发布版中的已知限制和常见部署注意事项。

- 前端当前以 Vite 服务方式运行在 Docker 中，生产部署可按需要改造为静态构建加 Web 服务。
- 大批量照片处理依赖 Worker 和 Redis，部署时需要确认 Worker 正常运行。
- 照片 EXIF 信息质量会影响自动定位，缺少坐标的照片需要人工补点。
- 公开仓库不包含示例照片和数据库快照，因此首次启动后需要自行创建项目并上传测试图片。
- NAS 或反向代理部署时，需要正确配置 `CORS_ORIGINS` 和 `VITE_ALLOWED_HOSTS`。

## Windows PowerShell npm.ps1 执行策略

在 Windows PowerShell 中运行 `npm` 时，可能出现无法加载或运行 `npm.ps1` 的执行策略提示。这通常是当前 PowerShell 会话禁止执行脚本，不是项目代码错误。

可以临时改用 Windows 命令入口：

```powershell
npm.cmd ci
npm.cmd run build
```

也可以只在当前 PowerShell 会话中放宽执行策略：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy RemoteSigned
```

该命令只影响当前 PowerShell 进程，关闭窗口后失效。
