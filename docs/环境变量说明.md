# 环境变量说明

## 数据库与 Redis

| 变量 | 说明 | 示例 |
| --- | --- | --- |
| `POSTGRES_DB` | PostgreSQL 数据库名 | `photo_map` |
| `POSTGRES_USER` | PostgreSQL 用户名 | `photo_user` |
| `POSTGRES_PASSWORD` | PostgreSQL 密码，生产必须修改 | `change_this_password` |
| `REDIS_URL` | Redis 连接地址 | `redis://redis:6379/0` |

## 服务端口

| 变量 | 说明 | 示例 |
| --- | --- | --- |
| `API_PORT` | API 暴露到宿主机的端口 | `8000` |
| `FRONTEND_PORT` | 前端暴露到宿主机的端口 | `5173` |
| `POSTGRES_PORT` | PostgreSQL 暴露端口，本机开发可用 | `5432` |
| `REDIS_PORT` | Redis 暴露端口，本机开发可用 | `6379` |

## 数据目录

| 变量 | 说明 |
| --- | --- |
| `HOST_DATA_DIR` | 宿主机持久化数据目录 |
| `DATA_DIR` | 容器内部数据根目录 |
| `ORIGINALS_DIR` | 原图目录 |
| `THUMBNAILS_DIR` | 缩略图目录 |
| `PREVIEWS_DIR` | 预览图目录 |
| `EXPORTS_DIR` | 导出目录 |
| `LOG_DIR` | 日志目录 |
| `REPORTS_DIR` | 报告目录 |
| `BACKUPS_DIR` | 备份目录 |

`HOST_DATA_DIR` 可以是 `./data`、`/volume1/docker/photo-map-v2/data` 或其他持久化路径。公开仓库不会包含这些运行数据。

## 访问与跨域

| 变量 | 说明 | 示例 |
| --- | --- | --- |
| `PUBLIC_BASE_URL` | 前端公网访问地址 | `https://photos.example.com` |
| `API_BASE_URL` | API 公网访问地址 | `https://api.photos.example.com` |
| `VITE_API_BASE_URL` | 前端构建/运行时 API 地址 | `https://api.photos.example.com` |
| `CORS_ORIGINS` | 允许访问 API 的前端来源 | `https://photos.example.com,http://localhost:5173` |
| `VITE_ALLOWED_HOSTS` | Vite 允许的额外 Host | `photos.example.com,api.photos.example.com` |

## 上传限制

| 变量 | 说明 | 示例 |
| --- | --- | --- |
| `MAX_UPLOAD_MB` | 单文件上传大小限制 | `50` |
| `MAX_UPLOAD_FILES` | 批量上传文件数量限制 | `999` |
| `VITE_MAX_UPLOAD_FILES` | 前端批量上传数量限制 | `999` |
| `VITE_UPLOAD_CONCURRENCY` | 前端上传并发数 | `3` |
