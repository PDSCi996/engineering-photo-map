# 目录结构说明

> English summary: This document explains the public repository layout and identifies runtime data that must stay out of GitHub.

本文说明公开仓库的目录结构，并列出不应提交到 GitHub 的运行数据。

```text
backend/                    FastAPI 后端服务
worker/                     Celery 媒体处理 Worker
frontend/                   Vue + Vite 前端
scripts/                    本地辅助脚本
tests/                      基础测试
tools/                      开发检查工具
docs/                       公开文档
docker-compose.yml          本地开发 Compose
docker-compose.synology.yml 群晖 Portainer 源码构建 Compose
docker-compose.image.yml    GHCR 镜像部署 Compose
.env.example                本地环境变量模板
.env.production.example     生产环境变量模板
.env.synology.example       群晖环境变量模板
```

## 不属于公开仓库的目录

以下内容可能包含真实照片、数据库、日志、部署信息或密钥，不应提交到 GitHub：

```text
.env
data/
logs/
reports/
backups/
.venv/
frontend/node_modules/
```

## 运行数据目录

应用运行后会在 `HOST_DATA_DIR` 指向的位置生成：

```text
originals/    原图
uploads/      上传暂存文件
previews/     预览图
thumbnails/   缩略图
exports/      导出文件
postgres/     数据库持久化数据
logs/         日志
reports/      检查报告
backups/      备份
temp/         临时文件
```

这些内容应由部署环境单独备份和保护，不进入公开仓库。
